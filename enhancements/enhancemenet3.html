Database Enhancement

Cloud-enable the Android Inventory Tracking App with Firebase Authentication and Cloud Firestore to support multi-user collaboration, real-time sync, and role-based access control.

  • Cloud Firestore
  • Security Rules
  • Realtime Sync
  • Integration
  • Android
  • Kotlin

Overview & Artifact Selection

The artifact I selected for this enhancement is an Android Inventory Tracking Application originally developed in CS 360: Mobile Architecture and Programming. The app was initially created as a standalone inventory manager, allowing users to add, edit, and delete items stored locally on their devices. It utilized basic on-device persistence to maintain inventory data and provided a straightforward interface for single-user use. However, the original version lacked multi-user functionality, device synchronization, and secure access controls, making it an ideal candidate for enhancement within the Databases category.

Why This Artifact

I chose this artifact for inclusion in my ePortfolio because it best demonstrates my ability to design, enhance, and secure data-driven mobile applications. This project highlights my growth in software architecture, database integration, and cloud-based development.

Implementation Details

The enhancement involved transforming the local storage system into a cloud-connected application by using Firebase Authentication and Cloud Firestore. These integrations enabled per-user data storage, real-time synchronization, and secure access from multiple devices. Additionally, I implemented a group inventory system that allows multiple users to collaborate on shared inventories. Each group included role-based permissions—Owner, Admin, and Member—enforced by carefully structured Firestore Security Rules to protect data integrity and privacy. The app also gained functionalities like pagination and per-document listeners, which improved performance and scalability by loading data in small batches and updating only modified records rather than reloading entire datasets.

Course Outcomes Alignment

Security Mindset & Privacy

This enhancement directly aligns with two key course outcomes. First, it demonstrates the ability to develop a security mindset that anticipates vulnerabilities and ensures privacy through secure software architecture. By implementing Firebase Authentication, I ensured that only verified users could access the database. The Firestore Security Rules I created followed the principle of least privilege, granting users access only to the data they owned or to the group inventories to which they were members. Each write operation was validated to enforce data type integrity and structural consistency, reducing the likelihood of unauthorized or malformed data.

Collaborative Environments

Second, this enhancement fully develops the outcome of employing strategies to build collaborative environments that enable diverse audiences to support organizational decision-making. By designing and implementing the group inventory system, I enabled teams of users to collaborate on shared datasets in real-time, each with clearly defined permissions and synchronized updates. This functionality supports organizational collaboration by ensuring data consistency across users and devices, making the app suitable for team-based inventory management in professional or business contexts.

Challenges & Resolutions

The process of enhancing this artifact helped me strengthen my skills in secure data handling, authentication workflows, and scalable mobile development. One of the significant lessons I learned was how to balance accessibility and security when designing cloud databases for collaborative applications. Implementing role-based access controls and writing Firestore rules that seamlessly integrate with client-side logic required careful planning and iterative testing to ensure seamless integration. Another challenge was optimizing performance for larger inventories. To maintain responsiveness without overwhelming the user interface or database, pagination and real-time listeners proved essential. Debugging asynchronous Firestore operations and resolving permission errors deepened my understanding of how cloud databases manage concurrency and synchronization. Throughout the enhancement process, I faced several significant challenges that enriched my understanding of mobile software design and cloud database management. A primary difficulty was configuring Firestore Security Rules to strike a balance between accessibility and protection effectively, ensuring that group owners, admins, and members had the appropriate level of access required careful logic structuring and repeated testing, as even minor rule errors could block valid operations or expose sensitive data. Debugging permission issues was often complex because Firestore's error messages are intentionally broad, necessitating a step-by-step approach to identify where access was being denied.

Future Work

One additional challenge I encountered during development was the lack of middleware functionality between the client and the database. Currently, the app communicates directly with Firestore, which, while efficient for rapid prototyping, limits flexibility and complicates the handling of multi-step operations or advanced security logic. Without a middleware layer, such as Firebase Cloud Functions, certain actions—like cascading deletions, role changes, or complex validations—must be managed entirely on the client side, increasing the potential for inconsistencies and security risks. A future upgrade could introduce middleware services to handle these operations on the server side, improving both reliability and security by centralizing logic and reducing client exposure to sensitive database operations.

Conclusion

Overall, this enhancement transformed the project from a simple local inventory manager into a robust, secure, and collaborative cloud-based system. It showcases my ability to design scalable applications that combine security, usability, and performance. This project represents a comprehensive demonstration of my growth as a developer capable of building and securing full-stack mobile solutions that function reliably in both individual and shared environments.
